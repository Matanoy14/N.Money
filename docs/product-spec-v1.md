# מפרט מוצר V1 — N.Money

---

## 1. מטרת המוצר

N.Money הוא מערכת ניהול פיננסי אישית, זוגית ומשפחתית בעברית — מרכזת בפלטפורמה אחת את כל תנועות הכסף, התקציב, ההכנסות, ההוצאות הקבועות, ההלוואות, הנכסים, והמטרות הפיננסיות. המוצר מאפשר לכל משתמש לראות את המצב הפיננסי שלו בבהירות מלאה, להבין לאן כל שקל הולך, ולהתקדם צעד אחד קדימה בכל פעם — בצורה פשוטה, פרימיום ומהימנה.

- הורים שולטים בהרשאות: הוספה בלבד / מצב מתקדם
- ילד אינו רואה הכנסות, הלוואות, נכסים — אלא אם הורה העניק גישה במפורש

## 5. מודל הנתונים המרכזי

### ישות קנונית אחת: `financial_movements`

כל ערך כספי במערכת — הכנסה, הוצאה, עסקה, הוצאה קבועה, העברה פנימית — הוא `financial_movement` אחד. אין ישויות DB נפרדות לסוגים שונים. ההבחנה מתבצעת דרך שדות `type` ו-`status`.

#### שדה `type`

| ערך | משמעות | השפעה על KPI |
|-----|---------|-------------|
| `income` | הכנסה | ✓ נכנס לסה"כ הכנסות ולתזרים נטו |
| `expense` | הוצאה | ✓ נכנס לסה"כ הוצאות ולתזרים נטו |
| `saving` | חיסכון / הפקדה למטרה | ✓ נכנס לצד ההוצאות לצורך תזרים |
| `transfer` | העברה פנימית בין חשבונות/ארנקות | **✗ אינו משפיע** על סה"כ הכנסות, הוצאות, או תזרים נטו |

> **הבהרה:** `type=transfer` משמש להעברות בין חשבונות בנק של אותו משתמש. הוא מוצג בפיד העסקאות אך מסוים מחישובי ה-KPI (הכנסות / הוצאות / תזרים נטו / מאזן).

#### שדה `status`

| סטטוס | משמעות |
|--------|---------|
| `planned` | תוכנן על ידי המשתמש — לא קרה עדיין |
| `expected` | צפוי מחזורי (הוצאה קבועה) — ממתין לאישור חודשי |
| `actual` | קרה בפועל — מאושר ומשפיע על כל החישובים |
| `pending_approval` | הוגש על ידי ילד / שינוי גדול — ממתין לאישור הורה |
| `simulation` | חישוב היפותטי — לא מופיע בנתוני אמת |
| `rejected` | נדחה על ידי מאשר |
| `cancelled` | בוטל על ידי המשתמש |

#### שדות מרכזיים

- `id` (uuid)
- `account_id` (חשבון משפחה/זוג)
- `user_id` (המשתמש שהוסיף)
- `type`: `income` | `expense` | `saving` | `transfer`
- `amount` (חיובי תמיד; סימן נגזר מ-`type` ומההקשר)
- `currency` (ברירת מחדל: ILS)
- `date`
- `description`
- `category`
- `sub_category`
- `payment_method`
- `status`
- `source`: `manual` | `voice` | `chat` | `recurring`
- `recurring_id` (קישור להוצאה קבועה, אם רלוונטי)
- `goal_id` (קישור למטרה, אם רלוונטי)
- `approved_by` (user_id של המאשר)
- `approved_at`
- `notes`
- `created_at`

### UX נפרד — בסיס נתונים משותף
- **עסקאות** — מסך ממוקד בהוצאות + הכנסות יומיומיות עם חיפוש, פילטור, הוספה/עריכה/מחיקה
- **הכנסות** — מסך ממוקד בתנועות מסוג `income`, עם פירוט לפי חבר חשבון ומקור

## 9. מפרט מודול-מודול

### עסקאות (Transactions)
- תצוגה של `financial_movements` מסוג `income` + `expense` + `saving` + `transfer` בסטטוס `actual` / `pending_approval`
- `transfer` מוצג בפיד אך מסוים מחישובי KPI
- פילטרים: סוג / חודש / קטגוריה / חבר חשבון / אמצעי תשלום
- חיפוש חופשי על תיאור וקטגוריה
- הוספה: פאנל צד — תיאור, סכום, תאריך, קטגוריה, אמצעי תשלום, הערות
- עריכה: פאנל זהה, טעון עם ערכים קיימים
- מחיקה: אישור לפני ביצוע
- פעולת ילד → `pending_approval`; מופיעה אצל הורה לאישור
- סיכום בר: הוצאות / הכנסות / מאזן לחודש (ללא transfer)

### הכנסות (Incomes)
- תצוגה ממוקדת: רק `financial_movements` מסוג `income`
- פירוט לפי חבר חשבון (בחשבון זוגי/משפחתי)
- פירוט לפי מקור: משכורת / פרילנס / שכ"ד / אחר
- הוספת הכנסה: כמו עסקה, עם שדה מקור
- סיכום: סה"כ הכנסות לחודש, לפי חבר, לפי מקור

### תקציב (Budget)
- הגדרת תקציב חודשי לפי קטגוריות
- תצוגת טבלה: מתוכנן / בפועל / הפרש / אחוז / סטטוס
- רמות: קטגוריה ראשית + תת-קטגוריות
- טאבים: הכל / קבועות / משתנות
- סטטוס: בגדר (ירוק) / קרוב (כתום) / חריגה (אדום)
- העתקת תקציב מחודש קודם
- עריכת ערכי תכנון בכל שורה
- תחזית סוף חודש בקצב הנוכחי

### הוצאות קבועות (Fixed Expenses)
- רשימת `financial_movements` בסטטוס `expected` — פריטים חוזרים חודשיים
- שדות: שם, קטגוריה, יום חיוב, אמצעי תשלום, סכום, סטטוס
- בתחילת כל חודש: אישור חודשי → `expected` הופך `actual`
- תזכורת בתוך המערכת לפני יום החיוב
- "תשלומים קרובים" — ווידג'ט מהיר
- הוספה / עריכה / מחיקה

### הלוואות והתחייבויות (Loans & Liabilities)
- שדות: שם, מלווה, סכום מקורי, יתרה, תשלום חודשי, ריבית, חודשים שנותרו, תאריך סיום משוער
- חישוב אוטומטי של שדות חסרים (למשל: סכום + ריבית + תשלום → מספר חודשים)
- לוח סילוקין (amortization): תשלומים עתידיים, ריבית מצטברת, יתרה חודשית
- סטטוס: פעיל / הסתיים / בהקפאה
- טיפ אסטרטגי: איזו הלוואה לסגור קודם
- הוספה / עריכה / מחיקה

### נכסים (Assets)
- רשימת נכסים: נדל"ן / רכב / קרן השתלמות / קופת גמל / פנסיה / תיק מניות / אחר
- שדות: שם, סוג, שווי נוכחי, עלות רכישה, שינוי %
- הון נטו = סה"כ נכסים − סה"כ חובות
- חלוקת שווי לפי קטגוריה
- הוספה / עריכה / מחיקה + עדכון שווי ידני

### מטרות (Goals)
- סוגי מטרות: אישית / זוגית / משפחתית / ילד ספציפי
- טווחי זמן: שבועי / חודשי / שנתי / תאריך יעד חופשי
- שדות: שם, סכום יעד, סכום שנחסך, הפקדה חודשית, תאריך יעד, עדיפות
- פרוגרס בר + % הושג + חודשים שנותרו
- קישור הפקדות חודשיות למטרה (goal_id)
- בתגיעה ל-100%: חגיגה ויזואלית + ארכוב / מטרה חדשה
- הוספה / עריכה / מחיקה

### מחשבונים (Calculators)

#### עצמאיים (Standalone)
- מחשבון משכנתא
- מחשבון חיסכון
- מחשבון הלוואה
- מחשבון פרישה

#### מחוברים לנתוני משתמש (Connected)
- "מה אם?" — סימולציות על הנתונים האמיתיים
- כל חישוב בסטטוס `simulation` — אינו נכנס לנתוני האמת

### מדריכים (Guides)
- מאגר מדריכים על מוצרים פיננסיים: משכנתא / קרן השתלמות / ביטוח / מניות / פנסיה / כרטיסי אשראי / חיסכון לילדים
- מבנה כל מדריך: מה זה / יתרונות / חסרונות / על מה לשים לב
- **לא ייעוץ אישי** — הסבר עקרוני בלבד
- פילטור לפי קטגוריה + חיפוש + מדריך מומלץ (featured)

## 10. מפרט הדשבורד

### KPI Cards (4 כרטיסים עליונים)

| כרטיס | נתון | צבע |
|-------|------|-----|
| הכנסות | actual income לחודש | ירוק (#00A86B) |
| הוצאות | actual expense+saving לחודש | **אדום (#E53E3E)** — תמיד; עוצמה עולה עם החריגה |
| תזרים נטו | הכנסות − הוצאות (ללא transfer) | ירוק אם חיובי / אדום אם שלילי |
| הון נטו | נכסים − חובות | כחול-זהב ניטרלי |

> **עיקרון צבע מוחלט:** הוצאות מוצגות תמיד בגוון האדום — לא בכחול. צבע כחול שמור לאלמנטי ממשק (כפתורים, לינקים, ניווט).

### עוצמת צבע דינמית
- ככל שהמצב חיובי יותר (חיסכון גבוה, מאזן טוב) → ירוק עמוק יותר
- ככל שהמצב שלילי יותר (חריגה, מאזן שלילי) → אדום מואמץ יותר
- בגדר ← → ממוצע ← → חריגה: מעבר הדרגתי

### גרפים
- **בר צ'ארט:** הכנסות (ירוק) מול הוצאות (אדום) — 6 חודשים אחרונים
- **דונאט:** חלוקת הוצאות לפי קטגוריה — חודש נבחר

### ווידג'טים
- **ציון בריאות פיננסית** — 0–100, מד עגול, V1: rule-based (ראו סעיף 10א')
- **קטגוריות מובילות** — 5 קטגוריות הוצאות הגדולות ביותר עם progress bar
- **עסקאות אחרונות** — 5 עסקאות אחרונות + קישור "לכל העסקאות"
- **תובנות** — strip אופקי: אזהרות / הצלחות / המלצות

### 10א' — ציון בריאות פיננסית (V1, Rule-Based)

הציון מחושב ממשקולות קבועות על 4 תחומים:

| תחום | משקל | חישוב |
|------|------|-------|
| חיסכון | 30% | (הכנסות − הוצאות) / הכנסות × 100 |
| תקציב | 30% | אחוז קטגוריות שנשארו בגדר |
| חובות | 25% | 1 − (סה"כ תשלומי הלוואות / סה"כ הכנסות) |
| מטרות | 15% | ממוצע % השגה של מטרות פעילות |

הציון הכולל = סכום משוקלל של 4 התחומים, מקסימום 100.
ב-V1 — אלגוריתם קבוע, ללא AI.

## 12. מפרט הבוט

### עקרון: ניטרלי מבחינת ספק
מנוע הבוט אינו נעול לספק מסוים בשלב זה. הממשק מוגדר כ-abstraction layer שיכול לעבוד עם כל מנוע LLM.

### מה הבוט עושה
1. **הזנת נתונים בצ'אט** — "שילמתי 200 שקל על קניות" → כרטיס עסקה לאישור לפני שמירה
2. **עזרה תפעולית** — "איך מוסיפים הוצאה קבועה?" → הסבר + ניווט
3. **הסבר מושגים** — "מה זה קרן השתלמות?" → הפניה למדריך הרלוונטי
4. **שאילתות מידע** — "מה הוצאתי על מסעדות החודש?" → שאילתה על הנתונים + תשובה

### מה הבוט לא עושה
- **לא נותן ייעוץ פיננסי אישי** — לא "קנה מניות X", לא "קח משכנתא Y"
- לא מבצע פעולות ללא אישור מפורש
- לא משתף מידע עם צדדים שלישיים

### ממשק
- פאנל צ'אט שנפתח מה-sidebar / FAB
- היסטוריית שיחה לפי session
- כפתור "הזן עסקה" לאחר זיהוי כוונת הזנה
